#### 目的
找出系统运行过程中内存的使用分布

#### 原理
建立了内存使用hash表
1. 内存申请时，hash表，记录此处内存申请代码所在的c文件名和行数
2. 内存释放时，删除链表对应的结点。
3. 统计当前申请
#### 使用方法：
##### 前置条件：
1. 至少需要SDK 250K以上的内存
2. 在引擎编译时，配置memory config->debug mem leak monitor 打开
3. 添加以下串口命令：

* 查看内存使用文件 gulite mem_used_file
```
    show_current_mem_dis();
```
