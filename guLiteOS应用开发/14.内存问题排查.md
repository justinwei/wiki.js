#### 目的
找出引擎可能存在的内存泄漏

#### 原理
建立了内存使用链表
1. 内存申请时，插入链表，记录此处内存申请代码所在的c文件名和行数
2. 内存释放时，删除链表对应的结点。

完成监控时，查询链表有哪些内存没有被释放。

#### 使用方法：
1. 只能在电脑上运行
2. 打开内存泄漏监控宏，realize_unit_mem.h
DEBUG_MEM_LEAK_MONITOR 1
3. make clean后从新编译
4. 注入监控开始和监控结束的代码

c语言
```
//启动监控
 mem_leak_start_monitor("name"); 

//统计监控
mem_leak_monitor_statics("name");        

```

js代码
```
//启动监控
gu_ui.mem_leak_start("name")
//统计监控
gu_ui.mem_leak_statics("name")     

```

5. 查看泄漏情况

运行时将日志输出到log文件里
```
 ./build/bin/guLiteOS_sim ../guliteos_liteapp_scanpen/app_launch_config.json > abc.log
```
打开abc.log
查找 total leak 如果有内存泄漏，可以看到有哪部分内存没有释放
