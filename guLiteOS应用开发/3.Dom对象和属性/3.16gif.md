## 属性说明
**只能定义在dom上面**
| 属性名称 | 描述 | 值类别 | 例子 |
|:-----|:----|:----|----|:----|
| src | 图片源 | 字符串，支持 本地文件，URL和base64三种格式  |```<gif src=”http://abc.com/123.jpg” /><gif src=“file:/123.jpg” /><gif src=“data:image/png;base64,iVBORw0KGgoAAAA...SAAAAABJRU5ErkJggg==” />``` |
| cache | 图片是否要缓存 | bool |```<gif cache="true" /> ``` |

| is_content_in_mem | 文件内容是否放在内存里，可以加快速度，但是消耗内存 | int |```<img is_content_in_mem="tue" /> ``` |
| alian | 相对定位 | alian |```<gif :alian='center' /> ``` |
| x | 定位，dom的x坐标 | px |```<gif x="50" /> ``` |

## src属性
src属性支持本地文件、网络路径、base64编码三种格式。

* 本地文件

```<gif src=“file:/123.gif” />```
必须以file:开头，APP运行目录为跟路径

* http的URL

```<gif src=”http://abc.com/123.gif” />```
URL地址，不推荐使用该方法，会造成设备端太多的http请求线程
如果页面只是一个大文件，可以使用这种方式

对于http格式，可以通过在dom添加 img_http_status_changed事件，获取图片下载的状态分别是
start，success，failed
例子：
```
layout.xml
    <gif :src="img_path" width="auto" height="auto"  
         @img_http_status_changed="img_changed() @ready="ready_cb()"/>     

script.js

    img_changed:function(status){
        console.log("image_changed status", status)
    }
    ready_cb:function(){
        console.log("ready_cb", status)
    }

```
* base64
服务器返回图片，建议使用data:base64格式，减少http请求，但建议每个文件小于30k
```<img src=“data:image/gif;base64,iVBORw0KGgoAAAA...SAAAAABJRU5ErkJggg==” />```
